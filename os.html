Processes & Threads

A process is a program in execution — an abstraction that includes code, data, stack, heap, and OS bookkeeping. Threads share the process address space but have their own stack and registers, enabling concurrency within a single process.

Process Control Block · States · Transitions
Core ▼

Threads & concurrency models
User threads · Kernel threads · M:N model
High freq ▼
  • User-level threads: managed by runtime, no kernel involvement — fast but one block = all block
  • Kernel threads: scheduled independently — blocking safe, but creation is costly
  • M:N (hybrid) model maps M user threads onto N kernel threads
  • POSIX pthreads use kernel threads; Java green threads were user-level

Context switching internals
Switch cost · voluntary vs preemptive
Medium ▼
  • Triggered by: timer interrupt, I/O wait, yield(), or higher-priority process arrival
  • Steps: save PC + registers → update PCB → select next process → restore PCB → resume
  • TLB flush on context switch adds significant overhead (unless hardware ASID is supported)
💬
Inter-process communication
Pipes · Shared memory · Message passing
Core ▼
  • Pipes: unidirectional, FIFO byte stream between related processes via file descriptors
  • Shared memory: fastest IPC — processes map same physical frames; needs explicit synchronization
  • Message passing: send/receive primitives — no shared state, safe but slower than shared memory
  • Sockets: IPC over network — used in client-server models (can also be used locally)

What is a process?

A process is an instance of a program in execution. Unlike a program (a passive entity on disk), a process is active — it has a program counter, registers, stack, heap, and associated OS metadata bundled in a Process Control Block (PCB).

PCB fields tested in GATE: Process ID, process state, program counter, CPU registers, CPU scheduling info, memory-management info, I/O status info, accounting info.

The OS uses the PCB to save and restore process state during context switches. Any GATE question asking "what is saved during a context switch" is really asking about PCB contents.

Process state transitions

The five-state process model is the most commonly tested in GATE. Know each transition trigger precisely:

New → Ready : admitted by OS (memory allocated) Ready → Running : dispatched by scheduler (CPU assigned) Running → Ready : preempted (timer interrupt / higher priority arrives) Running → Waiting : I/O request or event wait (voluntary block) Waiting → Ready : I/O completion / event occurs Running → Terminated : exit() called or unhandled exception
GATE trap: A process can go Running → Ready (preemption) but NEVER directly Waiting → Running. It must pass through Ready first. This distinction is asked almost every alternate year.

Threads: user-level vs kernel-level

The key distinction is where the thread table lives and who manages scheduling:

User-level threads: · Thread table in user space (managed by runtime library) · Kernel sees one process — not individual threads · One blocking syscall blocks ALL threads in the process · Fast creation/switching — no kernel mode transition needed Kernel-level threads: · Thread table in kernel space · Each thread scheduled independently by OS · Blocking safe — other threads in same process continue · Slower: every thread operation requires a kernel call

fork() — GATE numerical type

fork() creates an exact copy of the calling process. The number of processes after n fork() calls (if no process exits):

fork() called once → 2 processes (parent + child) fork() called twice → 4 processes fork() called n times → 2^n processes Example: 3 sequential fork() calls → 2^3 = 8 processes total
Variation tested: fork() inside a loop — count carefully. Each child also executes remaining loop iterations. Draw a process tree.
GATE 2024 2 marks Process states
Which of the following state transitions is NOT possible in a standard 5-state process model?
(A) Ready → Running   (B) Waiting → Running   (C) Running → Waiting   (D) Running → Ready
✅ Answer: (B) Waiting → Running is NOT possible. A process must return to the Ready state first before being dispatched to Running.
GATE 2023 1 mark Threads
In which threading model does a single blocking system call block all threads of the process?
(A) Kernel-level threading   (B) User-level threading   (C) Hybrid M:N model   (D) POSIX pthreads
✅ Answer: (B) User-level threading. The kernel only sees a single process — a blocking syscall from any user thread causes the entire process to block.
GATE 2022 2 marks Context switch
During a context switch from process P1 to P2, which of the following is saved as part of P1's context?
(I) Program Counter   (II) Stack Pointer   (III) Page table base register   (IV) Open file descriptors
✅ Answer: All four are saved — PC, SP, page table base register (CR3 in x86), and open file descriptor references are all part of the PCB.
GATE 2021 2 marks fork()
How many processes are created by the following code snippet (including the original process)?
for(i=0; i<3; i++) fork();
✅ Answer: 8 processes. Each fork() doubles the process count. After 3 forks: 2³ = 8 total processes (including the original).
GATE 2020 1 mark IPC
Which IPC mechanism provides the fastest communication between two processes on the same machine, assuming proper synchronization?
(A) Pipes   (B) Sockets   (C) Shared memory   (D) Message queues
✅ Answer: (C) Shared memory. After the initial setup, no kernel involvement is needed — processes communicate by reading/writing to a common physical memory region directly.
